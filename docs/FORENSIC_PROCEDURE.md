# Forensic журам (Chain of Custody)

Энэ систем нь нотлох баримтын бүрэн бүтэн байдлыг хадгалахын тулд дараах зарчмыг мөрдөнө.

## 1. Read-only (write-blocker)

Төхөөрөмж таниж мэдсэний дараа шинжилгээ эхлэхээс өмнө:

1. `blockdev --setro /dev/sdX` — блок төхөөрөмжийг kernel түвшинд зөвхөн унших болгох.
2. Mount хийх шаардлагатай бол `mount -o ro,noexec,nodev,noload` ашиглана.
3. Үйлдэл бүрийг `AuditLog`-д бичнэ (хэн, хэзээ, юу).

> Боломжтой бол **hardware write-blocker** ашиглахыг зөвлөнө. Software арга нь
> нэмэлт хамгаалалт болохоос hardware-ийн орлуулагч биш.

## 2. Imaging ба hash

- Анализыг аль болох бодит төхөөрөмж дээр биш, **forensic дүрс (image)** дээр хийнэ.
- Дүрс авахдаа `dd` (raw) эсвэл `ewfacquire` (E01) ашиглана.
- Дүрс авахаас өмнө болон дараа **SHA-256 (+ MD5)** hash тооцож, тэнцэж буйг баталгаажуулна.
- Эх төхөөрөмжийн hash = дүрсний hash байх ёстой (verify).

## 3. Chain of custody бүртгэл

`AuditLog` хүснэгтэд дараах үйлдлүүд автоматаар бүртгэгдэнэ:

- Device detect / attach
- Read-only тохируулсан
- Imaging эхэлсэн / дууссан (hash-тай)
- Scan эхэлсэн / дууссан
- Finding сэргээсэн / татаж авсан
- Тайлан үүсгэсэн

## 4. Findings ба бүрэн бүтэн байдал

- Сэргээсэн файл бүрийн SHA-256-ийг тооцож хадгална.
- Timestamp (MAC time: Modified/Accessed/Changed/Born) хадгалж timeline үүсгэнэ.
- Тайланд эх төхөөрөмжийн мэдээлэл, hash, шинжээчийн нэр, огноо бүгд орно.

## 5. Тайлан

Forensic тайлан (HTML/PDF) дараахийг агуулна:

- Хэргийн дугаар, шинжээч, огноо
- Төхөөрөмжийн мэдээлэл + дүрсний hash
- Олдсон ул мөрийн жагсаалт (устгагдсан файл, carved файл, recycle artifact)
- Timeline
- Chain-of-custody (audit) бүртгэл
